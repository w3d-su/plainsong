impact to editor correctness, user trust, or ability to enter Phase 2 safely.

## Immediate risk burn-down order

1. Specify the user-facing WYSIWYG release checklist (clean zero-width fold mechanism, edge snapping, `⌘⇧P` cycle entry + persisted layout mode, and a gate-rerun matrix) — the native interaction gates (IME, keyboard selection/copy/paste/accessibility, pointer/selection-edge) are now complete for the attribute-only hook.
2. Keep the opt-in actual IME (Zhuyin + Pinyin) harness and the pointer/selection gates runnable as regression coverage; rerun them before any user-facing WYSIWYG mode and against the final fold mechanism (R18).
3. Keep partial folded-span copy/paste policy documented and covered by EditorKit tests.
4. Keep user-facing WYSIWYG blocked until the release checklist is explicit and green.
5. Keep R17 as post-M5 polish; do not fold adjustable/native sidebar work into WYSIWYG gates.
6. Keep release hardening separate from Phase 2 feature work.
