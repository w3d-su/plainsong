# MDX Kitchen Sink Fixture

This fixture exercises Markdown mixed with ESM imports, JSX components, inline expressions,
frontmatter, fenced code, and ordinary prose. M5 preview should render the Markdown content,
show imports and exports as compact chips, and replace custom components with placeholder cards.

<Hero eyebrow="Native Markdown" title="Plainsong writes MDX without executing components" />

## Markdown Still Works

- **Bold** and *italic* spans remain ordinary Markdown.
- [Relative Markdown links](./kitchen-sink.md) should stay clickable.
- Inline math still renders: $a^2 + b^2 = c^2$.
- Emoji shortcodes are typed in source as `:sparkles:` and completed by M4.

> A blockquote inside an MDX document should preserve its preview styling and source-line anchor.

## Inline JSX And Expressions

Inline components like <Badge tone="success">Ready</Badge> should become safe inline placeholders.
Expressions such as {readingTime} and {new Date("2026-06-15").getFullYear()} should render as code chips,
not execute JavaScript.

Lowercase HTML can remain real HTML when sanitized:

<div className="note" data-kind="fixture">
  <strong>HTML island:</strong> this should not execute scripts or remote resources.
</div>

## Fenced Code

```tsx
export function Badge({ tone, children }: { tone: string; children: React.ReactNode }) {
  return <span data-tone={tone}>{children}</span>
}
```
